2 Configure Providers & API Keys
Enable each provider you want to use and paste its API key. You can have multiple providers active at once — the routing table (Step 3) controls which one is used for each feature.
OpenRouter

One key for 200+ models — GPT-4o, Claude, Llama and more.

OpenAI

Direct OpenAI API — GPT-4o, GPT-4o-mini, o1, o3.

HuggingFace

Free Inference API — Mistral, Llama, Falcon, Phi.

Ollama local

100% local, no API key needed. Runs on your machine.

LM Studio local

OpenAI-compatible local server, run any GGUF model.

3 Feature Routing — Which Provider & Model runs each feature?
For each app feature, choose the provider to use (or leave as "Default" to use the active provider). Then pick or type the model — click the field to see a dropdown of common models, or type any model name manually.
Feature Provider Model (type or select from dropdown)

Leaving "Provider" as Default and "Model" blank means the feature uses your Active Provider's default model.
